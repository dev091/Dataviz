# Platform Spec Audit

Use this skill to audit product and repo state against the governing platform contract.

## Workflow
1. Start with `docs/PRD.md`, `docs/LAUNCH_SCOPE.md`, `docs/ARCHITECTURE.md`, and `docs/TASK_CHECKLIST.md`.
2. Treat `docs/MASTER_ROADMAP.md` as the governing roadmap.
3. Review current code paths, API routes, UI surfaces, tests, and docs against the roadmap.
4. Report gaps honestly as `complete`, `partial`, or `missing`.
5. Prefer concrete repo evidence over claims.
